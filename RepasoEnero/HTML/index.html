GIT

Git es un sistema de control de versiones distribuido que transforma la colaboración en el desarrollo de software. Permite rastrear cambios en el código, coordinar el trabajo eficientemente y mantener un historial completo. Algunos puntos clave:

- Definición: Git es una herramienta que gestiona versiones de código, facilitando el desarrollo colaborativo.
- Uso: Es esencial para mantener un seguimiento de las modificaciones en el código y favorecer la colaboración.
- Ventajas: Facilita la gestión de proyectos y la integración de código con funciones como ramificación y fusión.

Comandos Básicos:
- git init: Inicia un nuevo repositorio.
- git add <archivo>: Agrega cambios al área de preparación.
- git commit -m "Mensaje": Realiza un commit con los cambios preparados.
- git push: Envía los cambios al repositorio remoto.
- git pull: Obtiene y fusiona los cambios del repositorio remoto.

Docker

Docker revoluciona la forma en que desarrollamos, implementamos y ejecutamos aplicaciones mediante la contenedorización. Permite empaquetar aplicaciones y sus dependencias en contenedores independientes, ofreciendo portabilidad y eficiencia. Algunos puntos clave:

- Definición: Docker utiliza contenedores para empaquetar y distribuir aplicaciones con todas sus dependencias.
- Uso: Facilita la creación de entornos de desarrollo consistentes y la implementación eficiente en diversos entornos.
- Ventajas: Ofrece portabilidad, eficiencia y escalabilidad en la implementación de aplicaciones.

Comandos Básicos:
- docker pull <imagen>: Descarga una imagen desde Docker Hub.
- docker build -t <nombre>: Construye una imagen a partir de un Dockerfile.
- docker run <imagen>: Ejecuta un contenedor basado en una imagen.
- docker ps: Muestra los contenedores en ejecución.
- docker stop <contenedor>: Detiene un contenedor en ejecución.
